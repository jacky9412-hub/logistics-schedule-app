# 物流排班查詢系統

這是一個手機優先的內部排班查詢原型，已依正式 Excel 資料改成「固定配置 + 異動覆蓋」模式。

## 核心規則

- Excel 下方每條正式路線後面的人名，代表該員工平常固定負責這個班別或路線
- 系統平常先依固定配置生成基礎班表
- 只有少數日期才透過請假、代班、支援去覆蓋原本固定班表
- 備員平常不綁固定路線，只在需要時代班
- 組長、行政、主管預設不綁固定路線
- 大夜班是固定勤務，主責人員為何x霖；休假時可由備員代班

## 正式資料來源

- 來源檔案：`C:\Users\User\Desktop\1111.xlsx`
- 已使用欄位：
  - 路線名稱
  - 員工姓名
  - 核定里程

## 角色分類

- 組長：2 人
- 備員：3 人
- 行政：1 人
- 主管：1 人，姓名為「主任」
- 其餘為運務員

## 目前功能

- 員工端
  - 查看今日固定班別或今日實際支援路線
  - 查看核定里程
  - 查看近期班表
- 組長端
  - 依固定配置批次生成日期區間班表
  - 登記請假與安排備員代班
  - 查看固定配置與今日實際配置差異
  - 透過主頁上方按鈕另開新視窗查看今日全體班表
- 行政端
  - 維護員工資料
  - 維護路線資料
  - 維護核定里程
  - 維護公司休假日
  - 可在白色文字框中直接手動輸入、刪除、修改休假日
  - 直接查看目前生效中的公司休假日清單
  - 手動新增的休假日會在清單中標示為「手動」
  - 可透過主頁上方按鈕查看今日全體班表
- 主管端
  - 查看全體固定配置
  - 查看異動紀錄
  - 具備最高維護權限

## 資料結構

- 員工欄位
  - 姓名
  - 角色
  - 固定班別
  - 固定路線
  - 支援線別
  - 是否備員
  - 是否可支援代班
  - 在職狀態
- 路線欄位
  - 類型
  - 路線名稱
  - 核定里程
- 排班欄位
  - 日期
  - 員工
  - 固定配置
  - 今日路線
  - 核定里程
  - 狀態
  - 假別
  - 來源：固定配置 / 異動覆蓋
  - 備註

## 排班邏輯

- 固定班表生成
  - 依固定配置展開指定日期區間
  - 自動略過週六、週日與公司休假日
  - 若某天已有異動覆蓋，保留異動資料，不強制洗掉
- 異動覆蓋
  - 可手動改單日路線、班別、狀態
  - 同一員工同一天重複排班會跳覆蓋確認
  - 同一路線同一天重複指派會跳警示
- 備員代班
  - 固定人員請假時，由備員接手路線
  - 大夜班也適用同樣流程
- 可支援代班運務員
  - 部分運務員雖不是正式備員，但可出現在代班人員下拉選單中
  - 平常仍保有原本固定路線與原本角色
- 主頁精簡
  - 主頁不再顯示「單日異動覆蓋」卡片
  - 主頁保留固定班表生成、請假與代班、資料維護與全體班表入口

## 核定里程說明

- 每條正式路線都有核定里程
- 核定里程顯示於：
  - 員工首頁
  - 班表列表
  - 路線維護表
  - 路線選單
- 用途：作為每日該路線的里程補貼依據

## 啟動方式

在專案資料夾執行：

```bash
npm start
```

然後開啟：

```text
http://localhost:3030
```

## 注意事項

- 目前仍是前端原型，資料儲存在瀏覽器 `localStorage`
- 若清除瀏覽器資料，內容會回到初始化資料
- 正式上線前仍建議補登入、後端 API 與資料庫

## 2026 休假日種子

- 系統已內建 2026 年台灣休假日種子
- 預設會自動略過：
  - 2026-01-01
  - 2026-02-16 至 2026-02-20
  - 2026-02-27
  - 2026-04-03
  - 2026-04-06
  - 2026-05-01 勞動節
  - 2026-06-19
  - 2026-09-25
  - 2026-10-09
- 週六、週日仍另外自動略過
- 後台仍可再手動新增或修正公司休假日
- 休假日設定中的白色文字框為可編輯區域，一行一個日期
- 上方休假日清單會依目前儲存內容同步更新，不再只顯示內建種子
